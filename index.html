Changing Fiscal Year in Analytics

Posted May 10th, 2006 by Mary Kay Bode
Analytics

iMIS analytics is shipped with the fiscal year set to the calendar year by default. This would be the same as having 0 or 1 in the AR/Cash setup for fiscal year.

To change the fiscal year for the transaction data for Analytics, the parameter just needs to be changed.

1) With the ‘Analytics Configuration’, set “ETL.FiscalYearBegins” to the first month of the fiscal year (1 to 12) and save. See Changing Analytics Configuration Options for instructions.

Instructing analytics to use financial transactions as the basis for sales history

Posted May 9th, 2006 by admin
Analytics

As shipped and by default, iMIS Analytics uses the Activity table for the source for sales history. However analytics supports using financial transactions (Trans) as an alternative source. ASI now recommends Trans as the preferred source for a number of reasons, but overall we feel that Trans based sales history is going to drive better and more timely information for all customers we are aware of.
